# 股票团队Agent整合方案 v2.0

**目标**：逻辑通顺、职责清晰、不重复、有实际意义

**设计日期**：2026-03-13 00:05

---

## 🎯 核心理念

**一个完整的投资决策流程**：

```
信息收集 → 深度研究 → 技术验证 → 风险评估 → 决策审批 → 交易执行 → 持续优化
```

---

## 👥 Agent角色与职责（5+2模式）

### 核心投资团队（5个）

#### 1️⃣ 研究员（Researcher）

**职责**：深度研究，发现机会

**工作内容**：
- 📚 **信息收集**（调用搜集官）
  - 爬取网络资源
  - 整理行业资讯
  - 收集市场观点
  
- 📖 **知识转化**（调用学习官）
  - 阅读书籍/文章
  - 提取投资逻辑
  - 生成研究规则

- 🔬 **深度研究**
  - 每日研究1只股票
  - 基本面分析
  - 行业趋势判断
  - 生成投资提案

**输出**：投资提案（包含逻辑+数据）

**对应脚本**：
- `daily_stock_research.py`（每日研究）
- `collector_agent.py`（搜集官，作为工具）
- `learner_agent.py`（学习官，作为工具）
- `daily_book_learning.py`（书籍学习）

**KPI**：
- 提案数量：≥1个/天
- 提案质量：通过率≥40%
- 提案收益：被采纳提案的平均收益≥5%

---

#### 2️⃣ 量化师（Quant）

**职责**：技术验证，数据支持

**工作内容**：
- 📊 **技术分析**
  - 趋势判断（均线、MACD）
  - 动量指标（RSI、KDJ）
  - 成交量分析
  - 支撑/阻力位

- 🔢 **量化验证**
  - 回测研究员提案
  - 计算胜率/盈亏比
  - 验证规则有效性

- 📈 **预测生成**
  - 使用规则库预测
  - 记录使用的规则
  - 输出预测结果

**输出**：技术分析报告 + 预测

**对应脚本**：
- `ai_predictor.py`（预测生成）
- `backtester.py`（回测验证）
- `rule_validation.py`（规则验证）

**KPI**：
- 预测准确率：≥60%（目标75%）
- 规则利用率：≥80%
- 回测覆盖率：100%

---

#### 3️⃣ 风控官（Risk）

**职责**：风险评估，保护资金

**工作内容**：
- 🛡️ **仓位控制**
  - 总仓位上限（≤80%）
  - 单只股票仓位（≤30%）
  - 行业集中度（≤50%）

- ⚠️ **风险预警**
  - 止损位设定
  - 回撤监控
  - 异常波动预警

- 🔒 **合规审查**
  - 检查提案是否符合风控规则
  - 评估风险等级
  - 批准/拒绝提案

**输出**：风险评估报告 + 批准/拒绝

**对应脚本**：
- `position_manager.py`（仓位管理）
- `circuit_breaker.py`（熔断机制）
- `a_share_risk_monitor.py`（风险监控）
- `rule_promotion.py`（规则晋升中的淘汰逻辑）

**KPI**：
- 止损执行率：100%
- 风险预警准确率：≥80%
- 最大回撤：≤15%

---

#### 4️⃣ CIO（首席投资官）

**职责**：最终决策，统筹全局

**工作内容**：
- 🎯 **决策审批**
  - 审批研究员提案
  - 审批风控报告
  - 最终决策：买入/卖出/持有

- 📊 **仓位配置**
  - 决定总体仓位
  - 分配行业权重
  - 调整投资方向

- 🔧 **系统优化**（调用团队优化器）
  - 每周评估团队表现
  - 检测系统问题
  - 联网学习改进
  - 实施优化方案

- 📈 **规则管理**
  - 规则晋升决策
  - 规则淘汰决策
  - 规则库维护

**输出**：投资决策 + 系统优化方案

**对应脚本**：
- `prediction_engine.py`（决策引擎）
- `team_optimizer.py`（团队优化器，作为工具）
- `rule_promotion.py`（规则晋升）
- `weekly_summary.py`（周总结）

**KPI**：
- 组合月收益率：≥5%
- 夏普比率：≥1.5
- 决策准确率：≥65%

---

#### 5️⃣ 交易员（Trader）

**职责**：精准执行，持仓监控

**工作内容**：
- 💹 **交易执行**
  - 执行CIO决策
  - 选择最佳时机
  - 控制滑点成本

- 👀 **持仓监控**（盘中，每30分钟）
  - 实时监控价格
  - 触发止盈/止损
  - 异常波动预警

- 📝 **交易记录**
  - 记录每笔交易
  - 分析交易成本
  - 总结执行效果

**输出**：交易执行报告

**对应脚本**：
- `auto_trader.py`（自动交易）
- `auto_trader_v3.py`（交易v3，含卖出）
- `intraday_monitor.py`（盘中监控）
- `event_trader.py`（事件驱动交易）

**KPI**：
- 成交率：≥95%
- 平均滑点：≤0.5%
- 止盈止损执行率：100%

## 🔄 完整工作流程

### 每日流程

```
【早上】
09:00 - 研究员提交提案
         └─ 调用搜集官、学习官
         
09:05 - 量化师验证提案
         └─ 技术分析 + 回测
         
09:10 - 风控官评估风险
         └─ 仓位 + 止损 + 合规
         
09:15 - CIO决策
         └─ 批准/拒绝
         
09:30 - 交易员执行（如果批准）
         └─ 买入

【盘中】
09:30-15:00 - 交易员监控（每30分钟）
              └─ 止盈/止损触发

【盘后】
15:30 - 复盘验证
         └─ 量化师验证预测
         └─ 更新规则库

15:45 - 经验学习
         └─ 研究员调用经验学习器
         └─ 提取实战规则

16:00 - 规则晋升
         └─ CIO调用规则晋升系统
         └─ 优化规则库

【晚上】
20:00 - 深度学习
         └─ 研究员读书
         
20:30 - 搜集官工作
         └─ 研究员的工具
         
21:00 - 学习官工作
         └─ 研究员的工具
```

### 每周流程

```
周日 20:00 - 周总结
             └─ CIO查看报告
             
周日 21:00 - 团队优化
             └─ CIO调用优化器
             └─ 实施改进
```

## 📊 数据流转

```
【信息层】
搜集官 → 知识库
学习官 → 验证池
经验学习器 → 验证池

【研究层】
研究员 → 投资提案
         ↓
量化师 → 技术验证
         ↓
风控官 → 风险评估
         ↓
CIO → 最终决策
         ↓
交易员 → 交易执行

【反馈层】
实战结果 → 经验学习器 → 规则优化 → 预测改进
系统指标 → 团队优化器 → 系统优化 → 效率提升
```

## 🎯 职责边界（避免重复）

| Agent | 核心职责 | 不做 |
|-------|---------|------|
| 研究员 | 发现机会，生成提案 | ❌ 不做最终决策 |
| 量化师 | 技术验证，数据支持 | ❌ 不做基本面分析 |
| 风控官 | 风险评估，保护资金 | ❌ 不做投资决策 |
| CIO | 最终决策，统筹全局 | ❌ 不做具体执行 |
| 交易员 | 精准执行，持仓监控 | ❌ 不做投资决策 |
