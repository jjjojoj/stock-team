## 🔺 第一层：选股层（动态标准）

### 股票候选池

**位置**: `config/stock_pool.md`

**范围**: A 股全部股票（动态筛选）

### 选股标准（动态进化）

**位置**: `config/stock_criteria.json`

**初始框架**:
```json
{
  "hard_filters": {
    "controller": ["央企", "省国资委", "市国资委"],
    "market_cap_max": 200,  // 亿
    "pb_max": 2.5,
    "exclude_st": true,
    "main_business_pct": 70
  },
  "soft_filters": {
    "roe_min": 10,
    "dividend_yield_min": 2,
    "industry_cycle": "low"
  }
}
```

**进化机制**:
- 每日 15:30 分析成功股票特征 → 更新标准
- 每周回顾 → 调整选股框架
- 每月总结 → 重构选股逻辑

**示例进化**:
```
发现：成功股票平均 PB=1.5，失败股票平均 PB=2.3
→ 调整：PB 标准从<2.5 改为<2.0

发现：成功股票 ROE 普遍>15%
→ 调整：ROE 要求从>10% 提高到>15%

发现：市值<100 亿成功率更高
→ 调整：市值上限从 200 亿改为 100 亿
```

### 选股规则库

**位置**: `learning/prediction_rules.json`

**规则分类**:
- `fundamental_rules` - 基本面规则
- `tech_rules` - 技术面规则
- `event_rules` - 事件驱动规则
- `sentiment_rules` - 情绪规则

**进化机制**:
- 每日 16:00 规则验证 → 统计成功率
- 置信度≥80% → 保留/加强权重
- 置信度<30% → 淘汰

### 选股流程

```
候选池 → 硬筛选 → 软筛选（打分） → 排序 → 前 20 只 → 观察池候选
```

**脚本**: `selector_v4.py`

## 🔺 第二层：预测层（观察池 + 持仓池）

### 观察池

**位置**: `config/watchlist.json`

**容量**: 15-20 只

**来源**: 选股层筛选

**管理**:
- 每日预测走势（涨跌/目标价/置信度）
- 置信度≥80% → 触发买入 → 进入持仓池
- 评分末尾淘汰 → 返回候选池
- 每日深度研究 1 股 → 补充新股

### 持仓池

**位置**: `config/positions.json`

**当前**: 7 只

**来源**: 观察池置信度≥80% 触发买入

**管理**:
- 每日预测走势（同观察池）
- 自动监控（涨跌停/熔断）
- 自动买卖时机决策
- 预测验证 → 反馈到预测系统

### 预测引擎

**位置**: `prediction_engine.py`

**预测依据**:
- 技术指标（RSI/MACD/均线）
- 基本面（PE/PB/ROE）
- 新闻情绪（正面/负面）
- 行业周期（低位/高位）
- 历史规律（类似走势统计）

**进化机制**:
- 每日 11:30 午盘反思 → 验证早盘预测 → 教训永久写入
- 每日 15:30 收盘复盘 → 验证全天预测 → 优化预测模型

## 🔺 第三层：交易层（完整买卖逻辑）

### 买入决策

**位置**: `auto_trader_v3.py`

**触发条件**（全部满足）:
- 预测置信度≥80%
- 无重大利空新闻
- 技术面突破信号
- 符合选股规则
- 仓位允许（单只<20%，总仓位<80%）

**仓位分级**:
| 置信度 | 仓位 |
|--------|------|
| 80-85% | 5%   |
| 85-90% | 10%  |
| 90-95% | 15%  |
| ≥95%   | 20%  |

### 卖出决策

**止损卖出**（强制）:
- 亏损达到 -8% → 立即卖出
- 突发重大利空 → 立即卖出
- 股票停牌 → 立即卖出
- 熔断触发 → 紧急减仓

**止盈卖出**（分级）:
- 盈利 +15% → 卖出 50%（锁定利润）
- 盈利 +20% → 卖出 80%
- 盈利 +30% → 全部清仓
- 移动止盈：盈利>10% 后，回撤 -5% 即卖出

**预测转空卖出**:
- 预测方向从"涨"转"跌"
- 置信度≥70%
- → 卖出持仓

**调仓卖出**:
- 每日 15:10 检查
- 发现更好机会（置信度更高）
- 卖出低置信度持仓
- 买入高置信度股票

**时间止损**:
- 买入后 5 个交易日无涨幅（<5%）
- → 卖出（资金效率）

### 风控系统

**熔断监控**:
- 每日 2 次：09:25 + 14:25
- 触发 → 紧急减仓 50%

**涨跌停监控**:
- 每日 2 次：09:25 + 14:25
- 触发 → 飞书预警

**仓位管理**:
- 单只股票≤20%
- 单行业≤40%
- 总仓位≤80%
- 动态调整（根据市场风险）

## 🔄 三层数据流动

### 正向数据流

```
选股层 → 预测层
   筛选股票 → 观察池

预测层 → 交易层
   置信度≥80% → 触发买入
   预测转空 → 触发卖出
```

### 反向反馈流

```
交易层 → 预测层
   交易结果 → 验证预测准确率

预测层 → 选股层
   预测准确率 → 验证选股质量
```

### 学习闭环

**选股层学习**:
- 每日 15:30 分析成功股票特征 → 更新选股标准
- 每周回顾 → 调整选股框架
- 每月总结 → 重构选股逻辑

**预测层学习**:
- 每日 11:30 午盘反思 → 即时优化
- 每日 15:30 收盘复盘 → 深度学习
- 教训永久写入 memory.md

**交易层学习**:
- 每笔交易记录 → 分析得失
- 每日 16:00 绩效汇报 → 统计胜率
- 每周回测 → 优化买卖逻辑

## ✅ 核心特点

### 1. 选股标准动态进化
- 初始框架：央企 + 市值<200 亿 + PB<2.5 + 非 ST
- 每日分析成功股票特征 → 调整标准
- 每周回顾 → 优化框架
- 每月总结 → 重构逻辑

### 2. 完整买卖逻辑
- 买入：置信度≥80% + 仓位分级
- 卖出：止损/止盈/预测转空/调仓/时间止损
- 风控：熔断/涨跌停/仓位限制

### 3. 三层独立学习
- 选股层：学习如何选好股票
- 预测层：学习如何预测走势
- 交易层：学习如何把握买卖时机

### 4. 数据互通 + 反馈闭环
- 正向：选股→预测→交易
- 反向：交易→预测→选股
- 每层数据反馈到其他层，共同优化

### 5. 无人值守 + 自我迭代
- 24 个 Cron 任务 24/7 自动运行
- 每日午盘反思 + 收盘复盘
- 每日规则验证 + 读书学习
- 系统持续进化，越用越聪明
