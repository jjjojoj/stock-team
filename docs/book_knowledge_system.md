### 阶段 3: 规则验证池（市场验证）⭐ **核心**

**位置**: `learning/rule_validation_pool.json`

**作用**: 书本知识不能直接用，必须经过市场验证！

**验证流程**:
1. 从书籍知识库提取知识点
2. 转化为可测试的规则
3. 在历史数据中回测
4. 在实盘中观察验证
5. 统计成功率

**验证规则结构**:
```json
{
  "rule_001": {
    "source": "book_001.point_001",
    "source_book": "股票作手回忆录",
    "rule": "价格沿最小阻力线运动",
    "testable_form": "突破 20 日高点后持有 5 日，胜率>50%",
    "category": "趋势",
    
    // 验证数据
    "backtest": {
      "samples": 100,
      "success_rate": 0.62,
      "avg_profit": 0.035,
      "avg_loss": -0.018,
      "profit_factor": 1.94
    },
    "live_test": {
      "samples": 20,
      "success_rate": 0.55,
      "started_at": "2026-03-15"
    },
    
    // 状态
    "status": "validating",  // validating/proven/rejected
    "confidence": 0.58,  // 综合置信度
    "created_at": "2026-03-10",
    "updated_at": "2026-03-20"
  }
}
```

**验证标准**:
| 指标 | 进入规则库要求 |
|------|--------------|
| 回测样本数 | ≥100 |
| 回测成功率 | ≥55% |
| 实盘样本数 | ≥20 |
| 实盘成功率 | ≥50% |
| 盈亏比 | ≥1.5 |
| **综合置信度** | **≥80%** |

### 阶段 4: 规则库（实战应用）

**位置**: `learning/prediction_rules.json`

**晋升机制**:
```
验证池规则 → 置信度≥80% → 规则库
                        ↓
                  优胜劣汰（持续）
```

**残酷淘汰机制**:
- 成功率<40% 且样本>10 → 权重 -20%
- 成功率<30% 且样本>20 → 标记移除
- 连续 5 次失败 → 立即移除
- **书籍规则无特权，一样淘汰**

---

### 阶段 5: 历史库（失败教训）

**位置**: `learning/rejected_rules.json`

**记录内容**:
- 规则内容
- 来源书籍
- 验证数据
- 淘汰原因
- 淘汰日期

**作用**:
- 避免重复错误
- 分析哪些书的知识不适用当前市场
- 知识进化历史

## 🔄 每日学习流程

### 20:00 每日读书

```bash
python3 scripts/daily_book_learning.py
```

**执行内容**:
1. 选择今日书籍（按书单顺序）
2. AI 阅读并提取 3-5 个核心观点
3. 存入书籍知识库
4. 转化 1-2 个观点为可验证规则
5. 加入规则验证池
6. 飞书汇报：今日学到的知识

---

### 15:30 规则验证（收盘后）

```bash
python3 scripts/rule_validation.py
```

**执行内容**:
1. 更新验证池中规则的表现
2. 统计成功率和置信度
3. 晋升达标规则到规则库
4. 淘汰失败规则
5. 飞书汇报：验证结果
